## 待辦項目

### 高優先順序

#### 1. AI 生圖功能（Image Generation）
- **描述**：支援 AI 圖像生成，可應用於角色視覺化、場景生成等
- **目標**：透過 LLM Provider 生成圖像並儲存至記憶系統
- **需求組件**：
  - [x] Image Generation Tool 實作（`tools/minimax_image.py`）
  - [ ] 支援 providers：DALL-E / Stable Diffusion / Flux 等
  - [ ] 主動發送圖片機制：AI 偶爾會根據心情或對話內容，主動生成並傳送圖片給使用者（限定 SU 權限使用）。
- **預估工時**：2-3 週

#### 2. 系統權限與隔離修復 (Security & Isolation)
- **描述**：修正近期發現的記憶串音與工具權限邊界問題
- **需求組件**：
  - [ ] **修復人格演化串音 Bug**：檢查歷史對話輸入，修正 `default` 助手使用了其他兩個 AI 的舊聊天紀錄進行反思的異常（且未活躍的助手不應觸發反思）。
  - [ ] **Bash 指令權限管控**：將 bash 指令組的權限嚴格限制為僅 SU (Super User) 可使用，一般使用者禁止觸發。
  - [ ] **背景話題搜尋隔離**：檢查背景話題搜尋邏輯，確保不會對一般使用者（非 SU）進行無效或越權的背景話題搜索。
  - [ ] **多使用者天氣與時間快取**：目前的快取機制是基於單一 SU 規劃的，需調整架構以安全支援多位使用者的獨立快取。

#### 3. UI 與體驗優化 (UI & UX)
- **描述**：增進介面操作便利性與多機器人識別度
- **需求組件**：
  - [ ] **Admin Bypass 登入開關**：在 Admin 帳號設定中加入 bypass 模式開關，啟用後可免除每次輸入帳號密碼，直接進入 Streamlit 管理介面。
  - [ ] **對話介面識別 (`chat.html`)**：在對話前端顯示當前正在講話的 AI 名稱（或大頭貼），以利在多角色情境下的視覺辨識。

#### 4. 多重 AI 群組對話 (Multi-Agent Group Chat)
- **描述**：支援多位不同人格的 AI 在同一個 Session 內共同聊天與互動。
- **目標**：使用者可動態將多位 AI 加入同一個聊天群組；系統需具備「中控路由（Central Orchestrator）」機制，由中控 LLM 決定哪位 AI 應該回應，並支援 AI 之間的互相對話。
- **需求組件**：
  - [ ] **群組 Session 管理**：修改 Session 架構以支援多 `character_id` 的綁定與上下文共享。
  - [ ] **中控 LLM 路由器**：實作中控分派層，分析對話上下文並決定是由哪位 AI 進行回覆。
  - [ ] **AI 互動觸發機制**：制定 AI 互相對話的規則與中止條件（避免無限對話迴圈）。
  - [ ] **群組管理 UI**：讓使用者能在介面上建立群組、邀請或移除特定的 AI 角色。
  - [ ] **預估工時**：4-6 週（複雜度較高，涉及底層編排重構）
