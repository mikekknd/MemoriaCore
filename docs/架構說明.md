## 系統概覽

MemoriaCore 是一套 AI 情境記憶引擎，以 FastAPI 作為後端核心，結合 BGE-M3 ONNX 嵌入模型實現雙軌向量檢索（稠密 + 稀疏），並搭配人格演進系統讓 AI 能持續自我觀察與反思。

支援多種前端接入：
- **Dashboard HTML** — 主要管理介面（`/static/dashboard.html`）
- **Streamlit UI** — 輔助管理後台（Port 8501）
- **Telegram Bot** — 聊天機器人接入
- **UnityClient** — 早期 Unity 範例程式仍在 repo 中；現行 auth/session 接入需先重新設計，不再維護舊指南文件

## 核心模組說明

### `api/routers/chat/` + `core/chat_orchestrator/`
對話編排層。`api/routers/chat/` 提供 REST / SSE / WebSocket 共用的 pipeline、execution 與單層 orchestration；`core/chat_orchestrator/` 提供雙層 Agent 協調、Router / Tool / Persona 分支，以及共用的 prompt/context/tool list 組裝。

### `core/llm_gateway.py`
LLM 路由器。支援多種任務類型，每種可獨立設定 Provider 與 Model：
`chat`、`expand`、`pipeline`、`compress`、`distill`、`ep_fuse`、`profile`、`persona_sync`、`persona_seed`、`background_gather`、`character_gen`、`router`、`group_router`、`translate`、`browser`

支援的 Provider：**Ollama（本地）**、**llama.cpp（本地）**、**OpenAI（雲端）**、**OpenRouter（雲端）**

### `core/core_memory.py`
記憶系統。以 SQLite WAL 模式儲存記憶區塊，實現混合向量搜尋（稠密 + 稀疏 + 時間衰減 + 重要性加權）。每個 LLM 模型擁有獨立的資料庫檔案（`memory_db_*.db`）。

### `core/memory_analyzer.py`
記憶分析器。在對話過程中偵測話題偏移，在背景觸發記憶管線（長文切分、摘要生成、圖譜縫合）。

### `core/personality_engine.py` + `core/persona_sync.py`
人格演化入口。早期 `ai_observe` / `ai_reflect` 流程仍存在；現行 Path D 同步由 `PersonaSyncManager` 逐角色、逐 face 寫入 `characters.json` 的 `evolved_prompt`，並將 trait snapshot 存入 `runtime/persona_snapshots.db`。

### `core/prompt_manager.py`
Prompt 統一管理器。從 `prompts_default.json` 讀取預設範本，支援使用者透過 Dashboard 即時覆寫，並在執行期將佔位符替換為實際內容。

### `core/storage_manager.py` + `core/storage/`
儲存層。`core/storage_manager.py` 保留 `StorageManager` public facade 與常數 re-export；實際 SQLite 操作依領域拆在 `core/storage/` repository mixins。主專案其他模組不得直接連 SQLite，需透過 `StorageManager` facade 新增或呼叫對應方法。

### `api/dependencies.py`
FastAPI 依賴注入中心。所有核心模組以 Singleton 形式初始化，透過 `Depends()` 注入各 Router。

### `api/session_manager.py`
Session 管理器。管理跨介面（HTML / Streamlit / Telegram / 外部 client）的對話狀態，支援自動過期清理。

## 前端介面

### Dashboard (`/static/dashboard.html`)
主要操作介面，透過 `http://localhost:8088/static/dashboard.html` 存取（需 FastAPI 運行）。Dashboard 以 iframe 掛載 chat、routing、prompts、bots、users、profile、persona tree、log viewer、DB viewer 等靜態頁。

### Streamlit UI (`app.py`, Port 8501)
輔助管理後台，保留設定、routing、prompts、DB、log、bots、character 等 admin-only 工具。對話主介面已轉移到 dashboard 靜態頁。

## 資料儲存

| 檔案 | 內容 |
|---|---|
| `runtime/memory_db_<model>.db` | 每模型獨立的記憶區塊（向量、重要性、時間戳）與核心認知 |
| `runtime/conversation.db` | 對話 Session 與訊息歷史 |
| `runtime/users.db` | 使用者、角色與登入 session metadata |
| `runtime/persona_snapshots.db` | Path D persona trait snapshot |
| `runtime/user_prefs.json` | 執行期設定：API Keys、LLM 路由設定、各種閾值 |
| `runtime/characters.json` | 角色設定與各角色 `system_prompt` / `evolved_prompt` |
| `prompts_default.json` | 所有任務的預設 Prompt 範本庫 |

### 關鍵閾值（`user_prefs.json`）

| 參數 | 說明 |
|---|---|
| `memory_threshold` | 向量相似度軟閾值（記憶召回） |
| `memory_hard_base` | 向量相似度硬閾值 |
| `shift_threshold` | 話題偏移偵測敏感度 |
| `cluster_threshold` | 記憶整合聚類閾值 |
| `reflection_threshold` | 觸發人格反思所需的觀察次數 |

## LLM 路由系統

每種任務類型可獨立設定 Provider 與 Model，透過 Dashboard Routing 頁或直接編輯 `user_prefs.json` 的 `routing_config` 欄位：

| 任務鍵 | 說明 | 建議 |
|---|---|---|
| `chat` | 即時對話（帶影子標籤） | 高參數量模型 |
| `expand` | User 意圖擴充 | 輕量快速模型 |
| `pipeline` | 一體化記憶管線 | 任意 |
| `compress` | 對話壓縮（編年史化） | 任意 |
| `distill` | 核心認知提煉 | 高品質模型 |
| `ep_fuse` | 情境概覽縫合 | 任意 |
| `profile` | 使用者畫像更新 | 任意 |
| `persona_sync` | Path D 人格同步 | 高品質模型 |
| `persona_seed` | Persona 初始 seed | 高品質模型 |
| `background_gather` | 背景話題摘要 | 輕量模型 |
| `character_gen` | 角色設定生成 | 高品質模型 |
| `router` | 意圖路由預處理 | 輕量快速模型 |
| `group_router` | 群組對話發話路由 | 輕量快速模型 |
| `translate` | TTS / reply 翻譯 | 輕量快速模型 |
| `browser` | Browser tool 任務 | 任意 |
